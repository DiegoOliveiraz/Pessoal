Após o cadastro bem-sucedido, o sistema deve armazenar os dados no localStorage e redirecionar o usuário para a página de confirmação (validacao.html).

VERSÃO 1.0 PRIORIDADE Alta (A)
ATORES ENVOLVIDOS Representante de Empresa, Sistema
ENTRADAS Nome Fantasia, Razão Social, CNPJ, e-mail, telefone, endereço, setor de atuação, senha
SAÍDAS Conta criada em fastwork_users.empresas, tela de confirmação
REGRAS DE NEGÓCIO CNPJ no formato XX.XXX.XXX/XXXX-XX; e-mail único entre empresas; nome 3–50 chars alfanuméricos; todos os campos obrigatórios
DESCRIÇÃO O sistema deve permitir que uma empresa crie uma conta na plataforma informando seus dados corporativos e de acesso. O CNPJ deve ser validado com máscara automática. Os dados são persistidos no localStorage sob a chave fastwork_users.empresas. Em caso de erro, o campo respectivo deve exibir borda vermelha e animação de shake.
