1. Flexbox vs. CSS Grid: A Diferença Fundamental

A diferença fundamental entre Flexbox e CSS Grid é a dimensionalidade:

 Flexbox trabalha em uma dimensão — organiza elementos em linha (row) ou em coluna (column), mas não os dois simultaneamente.
 CSS Grid, por outro lado, trabalha em duas dimensões, permitindo o controle tanto de linhas quanto de colunas ao mesmo tempo.

Exemplo de uso:

 Flexbox é ideal para alinhar itens em um menu horizontal, onde o foco está em distribuir espaço entre elementos de uma única linha.

  css
  nav {
    display: flex;
    justifycontent: spacebetween;
  }
  
 CSS Grid é indispensável para layouts completos de página, como a divisão entre cabeçalho, barra lateral, conteúdo e rodapé.

  css
  .layout {
    display: grid;
    gridtemplateareas:
      "header header"
      "sidebar main"
      "footer footer";
  }
  



 2. Alinhamento no Flexbox

As duas principais propriedades de alinhamento são:

 justifycontent → controla o alinhamento no eixo principal (mainaxis).
  Exemplo: justifycontent: center; centraliza os itens horizontalmente (se o flexdirection for row).

 alignitems → controla o alinhamento no eixo transversal (crossaxis).
  Exemplo: alignitems: center; centraliza os itens verticalmente (no caso de flexdirection: row).

Essas propriedades permitem alinhar os elementos com precisão dentro do contêiner flexível.



 3. Responsividade com CSS Grid

O CSS Grid permite criar layouts responsivos sem media queries complexas, usando:

 repeat() → repete um padrão de colunas.
 fr (fraction) → representa uma fração do espaço disponível.

Exemplo:

css
.grid {
  display: grid;
  gridtemplatecolumns: repeat(autofit, minmax(250px, 1fr));
  gap: 1rem;
}


Neste exemplo:

 autofit faz as colunas se ajustarem automaticamente ao espaço.
 minmax(250px, 1fr) define que cada coluna tenha no mínimo 250px e ocupe o espaço restante proporcionalmente.
  ➡️ Assim, o layout se adapta a diferentes larguras de tela sem precisar de media queries.



 4. O Sistema de Grid do Bootstrap

O Bootstrap Grid System é baseado em uma estrutura de containers, rows e columns:

 Container (.container ou .containerfluid) → define a área de conteúdo e aplica margens automáticas.
 Row (.row) → agrupa colunas e remove espaçamento lateral.
 Columns (.col) → dividem o espaço em 12 partes.

Exemplo:

html
<div class="container">
  <div class="row">
    <div class="colmd6">Coluna 1</div>
    <div class="colmd6">Coluna 2</div>
  </div>
</div>


O colmd6 significa que, em telas médias (md = ≥768px), a coluna ocupará 6 das 12 partes (50% da largura).
A responsividade é garantida pelos breakpoints, como sm, md, lg, xl, que adaptam o layout conforme o tamanho da tela.



 5. Componentes do Bootstrap e Acessibilidade

Exemplo: Navbar

A Navbar do Bootstrap é um componente que oferece responsividade (colapsa em um menu hambúrguer em telas menores) e acessibilidade, pois inclui atributos ARIA e navegação via teclado.

Vantagens:

 Responsiva sem necessidade de scripts personalizados.
 Compatível com leitores de tela e padrões WAIARIA.
 Economia de tempo e padronização de comportamento.

Fazer isso manualmente com apenas HTML e CSS exigiria muito mais código e testes de acessibilidade.



 6. Controle de Espaço no Flexbox

As propriedades flexgrow, flexshrink e flexbasis controlam o tamanho e comportamento dos itens:

 flexgrow → define quanto o item pode crescer quando há espaço disponível.
 flexshrink → define quanto o item pode encolher quando o espaço é reduzido.
 flexbasis → define o tamanho inicial do item antes da distribuição do espaço.

Valor padrão:
flex: 0 1 auto;
Isso significa:

 O item não cresce (flexgrow: 0),
 pode encolher (flexshrink: 1),
 E tem tamanho inicial automático (flexbasis: auto).



 7. Layouts Complexos com CSS Grid

Grid Areas permitem nomear partes do layout, facilitando a leitura e manutenção.

 gridtemplateareas → define o mapa do layout.
 gridarea → associa um elemento a uma dessas áreas nomeadas.

Exemplo:

css
.container {
  display: grid;
  gridtemplateareas:
    "header header"
    "sidebar main"
    "footer footer";
  gridtemplatecolumns: 200px 1fr;
}
.header { gridarea: header; }
.sidebar { gridarea: sidebar; }
.main { gridarea: main; }
.footer { gridarea: footer; }


Esse método torna o CSS mais semântico e organizado, ideal para layouts complexos.



 8. Customização do Bootstrap

Duas formas principais de customizar o Bootstrap:

1. Usando variáveis SASS
   Permite alterar cores, fontes e espaçamentos antes da compilação:

   scss
   $primary: 2c3e50;
   $fontfamilybase: 'Roboto', sansserif;
   @import "bootstrap";
   

2. Usando classes utilitárias personalizadas ou CSS sobrescrito
   Criar um arquivo custom.css e sobrescrever regras específicas, mantendo a estrutura do framework:

   css
   .btnprimary {
     backgroundcolor: ff6600;
     border: none;
   }
   

Essas abordagens mantêm a capacidade de atualização do Bootstrap.



 9. Alteração da Ordem Visual

A propriedade utilizada é order.

css
.item1 { order: 2; }
.item2 { order: 1; }


Essa propriedade existe tanto no Flexbox quanto no CSS Grid.
Ela é importante para responsividade, pois permite reorganizar visualmente os elementos (por exemplo, mover um menu para o topo em telas menores) sem alterar o HTML, o que também mantém a acessibilidade.



 10. Classes de Utilidade do Bootstrap

As classes de espaçamento seguem a sintaxe:


{property}{sides}{breakpoint?}{size}


 property: m (margin) ou p (padding)
 sides: t (top), b (bottom), l (left), r (right), x (eixo horizontal), y (vertical)
 breakpoint (opcional): sm, md, lg, xl
 size: 0 a 5 (ou auto)

Exemplos:

 pt3 → paddingtop de 1rem × 3 (nível 3).
 mxauto → margin horizontal automática (centraliza elementos).
 plg5 → padding em todos os lados nível 5, aplicado apenas em telas grandes (lg).

Essas classes tornam o layout mais rápido e consistente de configurar.
